Firebase Setup Guide for Digital GardenThis document outlines the steps to configure the serverless backend (Firebase) for your Digital Garden.1. Create ProjectGo to Firebase Console.Click Add project.Name it (e.g., digital-garden).Disable Google Analytics (optional, keeps it simple).Click Create.2. Register Web AppIn the project overview, click the Web (</>) icon.Name it (e.g., garden-site).Do not check "Firebase Hosting" yet (unless you know you want it).Click Register app.Copy the firebaseConfig object. You will need this for Step 5.3. Enable AuthenticationGo to Build > Authentication.Click Get Started.Sign-in method 1: Email/PasswordEnable "Email/Password".Click Save.Go to the Users tab and "Add user". This will be your Admin account.Sign-in method 2: Anonymous (Crucial for fixing permission errors)Click "Add new provider".Select Anonymous.Enable it and Save.Why? This allows visitors to "connect" to your database securely without creating an account, preventing "Insufficient Permission" errors.4. Enable Firestore DatabaseGo to Build > Firestore Database.Click Create Database.Select a location (e.g., nam5 or eur3).Start in Production mode.5. Configure Security RulesGo to the Rules tab in Firestore.Replace the code with the contents of firestore.rules (included in this repo).Click Publish.6. Connect Your CodeOpen index.html, admin.html, and article.html. Find the config block and replace it with your keys from Step 2:const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "your-project.firebaseapp.com",
  projectId: "your-project",
  // ... other keys
};
Troubleshooting"Missing or insufficient permissions":Check if Anonymous Auth is enabled in Firebase Console.Ensure your firestore.rules match the file provided.Ensure signInAnonymously is being called in index.html (we added this in the latest update).
